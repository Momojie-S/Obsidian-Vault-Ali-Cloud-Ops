# OpenClaw 代理配置调研

## 调研时间
2026-02-26（初版）| 2026-03-02（更新）

## 背景

需要让 OpenClaw 的 `web_fetch` 工具能够访问外网（如 GitHub 文档）。

## 最终方案：TUN 透明代理

2026-02-27 启用 Mihomo TUN 模式，系统级透明代理已正常运行。

详见 `网络配置.md#TUN 透明代理`

### 优点

- 所有程序自动走代理，无需单独配置
- 通过规则灵活控制直连/代理
- Clash 挂了可以快速切换节点或重启容器

### 注意事项

- 需要配置直连规则避免循环代理（代理节点服务器、本地服务等）
- SSH 端口 22 需要添加 `DST-PORT,22,DIRECT` 规则

## 历史方案（已废弃）

### 环境变量代理

在 `~/.openclaw/.env` 配置代理环境变量：

```bash
http_proxy=http://127.0.0.1:7890
https_proxy=http://127.0.0.1:7890
```

**废弃原因**：TUN 模式已启用，所有流量自动接管，环境变量代理不再需要。

### iptables 透明代理（已放弃）

详见 `网络配置.md#iptables 透明代理`

**放弃原因**：全量劫持风险大，Clash 节点不稳定会影响整个服务器网络。
