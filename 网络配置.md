# 网络配置

## DNS over HTTPS (DoH)

### 配置时间
2026-02-26

### 配置内容

编辑 `/etc/systemd/resolved.conf`：

```ini
[Resolve]
DNS=223.5.5.5#dns.alidns.com 119.29.29.29#doh.pub
DNSOverTLS=yes
```

> 注：国外 DoH（Cloudflare/Google）在国内可能被干扰，换成阿里 DNS 和腾讯 DNS 更稳定。

重启服务：

```bash
sudo systemctl restart systemd-resolved
```

### 验证

```bash
resolvectl status
# 确认显示 +DNSOverTLS
```

### 说明

- DoH 将 DNS 查询封装为 HTTPS 请求，防止 ISP 窥探和 DNS 污染
- 可独立使用，不依赖代理
- 配合 redsocks + iptables 透明代理时，DNS 也会走代理

### 相关命令

```bash
# 查看 DNS 状态
resolvectl status

# 测试 DNS 解析
resolvectl query google.com

# 查看 DoH 连接
resolvectl statistics
```

## TUN 透明代理（Mihomo）

### 配置时间
2026-02-27 启用

### 架构

使用 Mihomo (Clash Meta) 的 TUN 模式实现系统级透明代理：

```
所有流量 → TUN 接口 (Meta) → Mihomo → 按规则分流
                                    ├── DIRECT (直连)
                                    └── Proxy 节点
```

### Docker 部署

```bash
# 容器名称: clash
# 镜像: metacubex/mihomo:latest
# 网络模式: host
# 配置挂载: /home/momojie/env/docker/clash-conf/config.yml

docker run -d \
  --name clash \
  --restart always \
  --network host \
  -v /home/momojie/env/docker/clash-conf/config.yml:/root/.config/mihomo/config.yaml:ro \
  metacubex/mihomo:latest
```

### TUN 配置

```yaml
tun:
  enable: true
  stack: system
  auto-route: true
  auto-detect-interface: true
  dns-hijack:
    - any:53
    - tcp://any:53
```

### 直连规则（服务器专用）

自定义规则通过脚本维护，**不要手动编辑 config.yml**（会被覆盖）。

**配置脚本**：`/home/momojie/env/docker/clash-conf/update-clash-config.sh`

**更新流程**：
1. 编辑脚本中的 `CUSTOM_RULES_FILE` 部分
2. 运行 `./update-clash-config.sh --tun` 重新生成配置
3. 重启容器 `docker restart clash`

**当前自定义规则**：

```yaml
rules:
  # ===== 自定义规则（服务器专用）=====
  # 代理节点服务器直连（避免循环代理）
  - DOMAIN-SUFFIX,mojcn.com,DIRECT
  - DOMAIN-SUFFIX,cnmjin.net,DIRECT
  - DOMAIN-SUFFIX,cnmjcn.cyou,DIRECT
  # GLM 大模型直连
  - DOMAIN-SUFFIX,bigmodel.cn,DIRECT
  - DOMAIN-SUFFIX,zhipuai.cn,DIRECT
  # DoH 直连
  - DOMAIN-SUFFIX,alidns.com,DIRECT
  - DOMAIN-SUFFIX,doh.pub,DIRECT
  - IP-CIDR,223.5.5.5/32,DIRECT,no-resolve
  - IP-CIDR,119.29.29.29/32,DIRECT,no-resolve
  # 企业微信直连
  - DOMAIN-SUFFIX,qyapi.weixin.qq.com,DIRECT
  # GitHub 直连（服务器可直连）
  - DOMAIN-SUFFIX,github.com,DIRECT
  - DOMAIN-SUFFIX,githubusercontent.com,DIRECT
  - DOMAIN-SUFFIX,githubassets.com,DIRECT
  - DST-PORT,22,DIRECT
  # OpenClaw 本地服务直连
  - IP-CIDR,127.0.0.1/32,DIRECT
```

### 问题排查

**SSH 连接 GitHub 超时**
- 原因：TUN 劫持所有流量，SSH 端口 22 走了代理
- 解决：添加 `- DST-PORT,22,DIRECT` 规则
