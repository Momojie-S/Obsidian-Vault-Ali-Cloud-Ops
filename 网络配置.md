# 网络配置

## DNS over HTTPS (DoH)

### 配置时间
2026-02-26

### 配置内容

编辑 `/etc/systemd/resolved.conf`：

```ini
[Resolve]
DNS=223.5.5.5#dns.alidns.com 119.29.29.29#doh.pub
DNSOverTLS=yes
```

> 注：国外 DoH（Cloudflare/Google）在国内可能被干扰，换成阿里 DNS 和腾讯 DNS 更稳定。

重启服务：

```bash
sudo systemctl restart systemd-resolved
```

### 验证

```bash
resolvectl status
# 确认显示 +DNSOverTLS
```

### 说明

- DoH 将 DNS 查询封装为 HTTPS 请求，防止 ISP 窥探和 DNS 污染
- 可独立使用，不依赖代理
- 配合 redsocks + iptables 透明代理时，DNS 也会走代理

### 相关命令

```bash
# 查看 DNS 状态
resolvectl status

# 测试 DNS 解析
resolvectl query google.com

# 查看 DoH 连接
resolvectl statistics
```

## 透明代理（已放弃）

### 尝试方案

使用 redsocks + iptables 实现全局 TCP 透明代理：

```
任意 TCP 请求 → iptables 劫持 → redsocks → Clash → 代理出墙
```

### 放弃原因

1. **全量劫持风险大**：iptables 工作在网络层，无法按域名分流，只能按 IP/端口
2. **Clash 节点不稳定**：代理节点经常变化，一旦节点挂掉，整个服务器网络都受影响
3. **循环依赖问题**：Clash 连接代理服务器的出站流量也会被 iptables 劫持，需要复杂的排除规则
4. **维护成本高**：Clash 配置更新后可能需要调整 iptables 规则

### 替代方案

使用环境变量，让需要代理的程序（如 OpenClaw 的 web_fetch）走代理：

```bash
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
```

优点：
- 只有指定程序走代理，其他服务不受影响
- Clash 挂了只影响代理相关功能
- 无需维护 iptables 规则
